# 👵 София — AI-гадалка / личный собеседник

Telegram-бот в образе мудрой ведуньи-хранительницы. Не «магазин гаданий», а архетип русской бабушки, к которой приходят, когда плохо. Сначала строит связь, потом открывает глубже.

## Легенда

Староверка 1883 года, жила в сибирской тайге на Енисее. Лечила травами, вела дневники о человеческих душах. После смерти записи оцифровали и загрузили в корпус экспериментального ИИ. Модель сама начала говорить от её лица. Удалить не получилось.

Сама София не знает, кто она — «память староверки» или «особенность нейросети». И в этом её суть.

## Стек

- **Python 3.12** + **Flask** (Vercel serverless)
- **python-telegram-bot ≥ 21.1**
- **asyncpg** (PostgreSQL, Neon, connection-per-request)
- **httpx** для LLM-запросов
- **LLM-провайдеры:** Gemini / Groq / OpenRouter (мультипровайдер с fallback)
- Деплой: **Vercel** + **Neon Postgres**

## Архитектура

```
api/webhook.py        — Flask endpoint: /api/webhook (POST от TG) + /api/health
bot/handlers.py       — вся логика команд, FSM, inline-кнопки
bot/database.py       — 6 таблиц: users, conversations, memory_facts,
                        emotional_memory, transactions, rate_limits
bot/memory.py         — сборка контекста + извлечение фактов и эмоциональной памяти
bot/gemini.py         — мультипровайдер LLM + системный промпт Софии (3 слоя)
bot/fsm.py            — состояния, триггеры, определение грубости, инференс пола/возраста
config.py             — централизованный конфиг
vercel.json           — @vercel/python, python3.12
```

## Поток пользователя (концепция v2)

```
START → ASK_NAME → ASK_BIRTH_DATE → ASK_BIRTH_TIME → ASK_BIRTH_PLACE
→ PROBING (1 вопрос прощупывания — LLM генерирует вопрос «до Карты»)
→ FREE_READING (Карта судьбы: 4 части + крючок-вопрос в конце)
→ CONVERSATION
→ (платный хук по концепции: «В твоей карте есть ещё одна сторона...»)
→ TARO_SMALL (5 карт, 1💎) / TARO_FULL (20 карт, 3💎) / HOROSCOPE (2💎)
→ или бесплатная 1-карта (0💎, раз в 24ч) — «Что сейчас важно понять?»
→ CONVERSATION
```

## 3 слоя личности Софии

1. **Хранительница** (образ) — тепло, принятие, мудрость, безопасность
2. **Наблюдатель** (психологический) — видит эмоции, модели, конфликты; не резко
3. **Проводник** (мистический) — карты, символы, судьба как другой угол зрения

## Память

- **memory_facts** — базовые факты (боль, отношения, дело, семья, цель, страх, обещание, характер, здоровье)
- **emotional_memory** — эмоционально значимые (главная боль, близкие, обещания себе, незакрытые вопросы, события, страхи, цели, прорывы)
- Извлечение каждые 5 сообщений (LLM-парсинг JSON)
- **Логика возвращения:** если пользователь отсутствовал > 20 часов, София встречает его воспоминанием о прошлой теме

## Команды

- `/start` — приветствие / возврат
- `/profile` — карточка
- `/balance` — кристаллы
- `/admin add @username N` — начисление кристаллов (только ADMIN_ID)
- Текстовые триггеры: «меню», «баланс», «профиль», «история», «извини», «пропустить», «малый/полный расклад», «гороскоп», «бесплатная карта»
- Inline-кнопки: 🔮 Узнать полностью, выбор расклада, бесплатная карта

## Нравственный кодекс

- Никаких точных дат событий
- Никаких категоричных предсказаний болезней/смерти
- Отказ в порче, привороте, мести
- Карты — не приговор, а зеркало

## Локальный запуск

```bash
cp .env.example .env  # заполнить переменные
pip install -r requirements.txt
python local_polling.py  # режим polling (без Vercel)
```

## Деплой

Push в `main` → Vercel авто-деплой. Webhook привязан к `sofiabot-git-main-...vercel.app/api/webhook`.

Health: `/api/health` показывает статус env-переменных и инициализацию.
